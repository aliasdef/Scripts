# Automated Linux Backup & Rotation Script

Bash-скрипт для автоматического создания резервных копий рабочих директорий с интегрированной функцией ротации (очистки устаревших архивов) и логированием всех этапов работы.

## Основной функционал

*   **Архивация на лету**: Сжатие целевой папки в формат `tar.gz` с сохранением структуры.
*   **Динамические таймштампы**: Автоматическое присвоение уникальных имен архивам по маске `YYYY-MM-DD_HH-MM-SS`.
*   **Интеллектуальная ротация (Retention Policy)**: Автоматическое сканирование бэкап-директории и удаление архивов старше 14 дней для предотвращения переполнения диска [🎯].
*   **Валидация данных**: Встроенная проверка существования целевой папки перед стартом процесса для предотвращения пустых дампов.

## 🚀 Инструкция по развертыванию

### 1. Клонирование и настройка прав доступов
Скачайте скрипт и выдайте ему права на исполнение.
```bash
chmod +x backup.sh
```

### 2. Ручной запуск
Для принудительного создания резервной копии выполните:
```bash
./script.sh
```

### 3. Автоматизация через Планировщик Cron

Для настройки регулярного резервного копирования по расписанию добавьте задачу в `crontab`:
```bash
crontab -e
```
Добавьте следующую строчку в конец файла для запуска скрипта **каждый понедельник в 18:00**:
```text
0 18 * * 1 /путь_к_скрипту/backup.sh
```

## Переменные среды окружения

Перед запуском вы можете кастомизировать пути внутри скрипта
*   `DIR` — абсолютный путь к директории, данные из которой нужно забэкапить.
*   `BACKUP_DIR` — путь к хранилищу готовых архивов.


##Безопасность и ИБ-аудит

Данный скрипт разработан в рамках построения безопасной инфраструктуры. Процесс удаления устаревших копий через связку `find -mtime +14 -delete` работает локально и изолированно, исключая риски инъекции команд со стороны непривилегированных пользователей.
